## Case Study 1: Code Review Pipeline

**Goal**: Review pull request with multiple quality checks in parallel.

**Workflow**: Fan-out → Gate → Sequential fixes

```python
# workflow.py
async def code_review_pipeline(pr_number, repo):
    """Multi-agent code review with parallel analysis."""
    
    # STAGE 1: Parallel analysis (Fan-out)
    analyses = await fanout_workflow([
        f"Review code style and formatting for PR #{pr_number}",
        f"Check for security vulnerabilities in PR #{pr_number}",
        f"Analyze test coverage for PR #{pr_number}",
        f"Review architecture/design patterns in PR #{pr_number}"
    ], agent_id="reviewer", timeout=90)
    
    # Quality gate: All must complete
    if analyses["success_rate"] < 1.0:
        return {
            "status": "failed",
            "reason": f"{analyses['failed']} review(s) failed to complete",
            "retry": True
        }
    
    # STAGE 2: Synthesize findings
    summary = await sessions_spawn({
        "task": f"Synthesize these code reviews into actionable summary:\n{format_reviews(analyses['results'])}",
        "agentId": "synthesizer",
        "timeoutSeconds": 60
    })
    
    # STAGE 3: Generate fix suggestions (if issues found)
    issues = extract_issues(summary["output"])
    if issues:
        fixes = await sessions_spawn({
            "task": f"Generate fix suggestions for these issues:\n{issues}",
            "agentId": "coder",
            "timeoutSeconds": 120
        })
        summary["fixes"] = fixes["output"]
    
    return {
        "status": "complete",
        "summary": summary["output"],
        "issues_count": len(issues),
        "approvable": len(issues) == 0
    }

# Execution
result = await code_review_pipeline(pr_number=42, repo="myapp")
# Returns: {status, summary, issues_count, approvable}
```

**Key Patterns**:
- 4 parallel reviewers (fan-out)
- 100% completion gate before synthesis
- Conditional fix generation (only if issues found)
- Total time: ~3min vs 8min sequential

## Case Study 2: Documentation Generator

**Goal**: Generate comprehensive documentation from codebase.

**Workflow**: Map → Parallel → Reduce → Polish

```python
async def generate_docs(repo_path, output_dir):
    """Generate structured documentation from code."""
    
    # MAP: Identify components to document
    components = await sessions_spawn({
        "task": f"Scan {repo_path} and list all public APIs, modules, and key functions that need documentation",
        "agentId": "analyzer",
        "timeoutSeconds": 60
    })
    
    component_list = parse_component_list(components["output"])
    
    # PARALLEL: Document each component
    docs = await fanout_workflow([
        f"Generate documentation for {comp['name']}: {comp['path']}"
        for comp in component_list
    ], agent_id="tech-writer", timeout=120)
    
    # REDUCE: Combine into structure
    combined = await sessions_spawn({
        "task": f"Organize these component docs into cohesive documentation:\n{combine_docs(docs['results'])}",
        "agentId": "synthesizer",
        "timeoutSeconds": 90
    })
    
    # POLISH: Add examples and cross-references
    final = await sessions_spawn({
        "task": f"Add usage examples and cross-references to:\n{combined['output']}",
        "agentId": "tech-writer",
        "timeoutSeconds": 120
    })
    
    # Write to disk
    await write_file(f"{output_dir}/README.md", final["output"])
    
    return {"components": len(component_list), "output": final["output"], "path": output_dir}

# Execution
result = await generate_docs("./src", "./docs")
# Generates: Structured README with examples for all components
```

**Key Patterns**:
- Map-Reduce pattern (discover → document each → combine)
- Parallel docs for speed
- Sequential polish for coherence
- Checkpoint at each stage

## Case Study 3: Incident Response

**Goal**: Diagnose and respond to production incident.

**Workflow**: Parallel investigation → Root cause → Action plan → Communication

```python
async def incident_response(alert_data, severity="high"):
    """Multi-agent incident response workflow."""
    
    context_mgr = ContextManager(state_file="./incident-state.json")
    
    # PHASE 1: Parallel investigation
    investigations = await fanout_workflow([
        f"Check logs for errors related to: {alert_data['service']}",
        f"Analyze metrics for anomaly at {alert_data['timestamp']}",
        f"Check recent deployments affecting {alert_data['service']}",
        f"Review error patterns in traces"
    ], agent_id="ops-analyst", timeout=60)
    
    context_mgr.add("investigation", "parallel", investigations)
    
    # PHASE 2: Root cause analysis (if investigations succeeded)
    if investigations["success_rate"] >= 0.75:
        root_cause = await sessions_spawn({
            "task": f"Determine root cause from these findings:\n{investigations['results']}",
            "agentId": "sre",
            "timeoutSeconds": 90
        })
        context_mgr.add("sre", "root-cause", root_cause, decisions=[root_cause["output"]])
    else:
        # Escalate to human if too many failures
        return await escalate_to_oncall(alert_data, investigations)
    
    # PHASE 3: Generate action plan
    action_plan = await sessions_spawn({
        "task": f"Generate immediate and long-term fixes for: {root_cause['output']}",
        "agentId": "sre",
        "timeoutSeconds": 60
    })
    
    # PHASE 4: Draft communications
    if severity in ["high", "critical"]:
        comms = await sessions_spawn({
            "task": f"Draft incident communication for stakeholders. Root cause: {root_cause['output']}",
            "agentId": "communicator",
            "timeoutSeconds": 45
        })
        action_plan["communication"] = comms["output"]
    
    return {
        "incident_id": alert_data["id"],
        "root_cause": root_cause["output"],
        "action_plan": action_plan["output"],
        "communication": action_plan.get("communication"),
        "duration": context_mgr.total_duration()
    }

# Execution
result = await incident_response({"service": "api", "timestamp": "2026-02-20T10:00:00Z"})
# Returns: Root cause, fixes, and stakeholder communication
```

**Key Patterns**:
- Time-critical parallel investigation
- 75% success gate (tolerates 1 failure in 4)
- Conditional escalation path
- Context tracking for audit trail
- Severity-based branching (communications only for high/critical)

## Case Study 4: Research Synthesis

**Goal**: Deep research with fact-checking and synthesis.

**Workflow**: Recursive research → Fact-check → Synthesize → Validate

```python
async def deep_research(topic, depth=2):
    """Multi-depth research with verification."""
    
    findings = []
    
    # LEVEL 1: Initial broad research
    initial = await sessions_spawn({
        "task": f"Research {topic} and identify 5 key sub-topics",
        "agentId": "researcher",
        "timeoutSeconds": 120
    })
    
    subtopics = extract_subtopics(initial["output"])
    
    # LEVEL 2: Parallel deep dives (if depth > 1)
    if depth > 1:
        deep_research_tasks = [
            f"Deep research on {sub}: key facts, sources, controversies"
            for sub in subtopics[:3]  # Limit to prevent explosion
        ]
        
        deep_results = await fanout_workflow(
            deep_research_tasks,
            agent_id="researcher",
            timeout=180
        )
        
        findings.extend(deep_results["results"])
    
    # FACT CHECK: Verify key claims
    claims = extract_claims(findings)
    verifications = await fanout_workflow([
        f"Verify this claim with sources: {claim}"
        for claim in claims[:10]  # Check top 10 claims
    ], agent_id="fact-checker", timeout=60)
    
    # SYNTHESIZE: Combine into coherent report
    report = await sessions_spawn({
        "task": f"Synthesize verified research into structured report:\n{combine_with_verifications(findings, verifications['results'])}",
        "agentId": "synthesizer",
        "timeoutSeconds": 120
    })
    
    return {
        "topic": topic,
        "depth": depth,
        "subtopics": len(subtopics),
        "claims_verified": len(verifications["results"]),
        "report": report["output"],
        "confidence": calculate_confidence(verifications)
    }

# Execution
result = await deep_research("Quantum computing applications in cryptography", depth=2)
# Returns: Verified research report with confidence score
```

**Key Patterns**:
- Recursive depth control (limit subtopics to prevent explosion)
- Separate fact-checking phase with parallel verification
- Confidence scoring based on verification results
- Token-conscious limiting (top 3 subtopics, top 10 claims)

## Case Study 5: Microservice Generator

**Goal**: Generate production-ready microservice from specification.

**Workflow**: Architecture → API design → Implementation → Tests → Documentation

```python
async def generate_microservice(spec):
    """Complete microservice generation pipeline."""
    
    ctx = ContextManager(state_file=f"./{spec['name']}-state.json")
    breaker = CircuitBreaker(failure_threshold=2)
    
    # STAGE 1: Architecture design
    arch = await breaker.call(
        sessions_spawn,
        {"task": f"Design architecture for: {spec['requirements']}", "agentId": "architect", "timeoutSeconds": 120}
    )
    ctx.add("architect", "design", arch, decisions=["tech_stack", "data_model"])
    
    # STAGE 2: API specification
    api = await sessions_spawn({
        "task": f"Generate OpenAPI spec based on architecture:\n{arch['output']}",
        "agentId": "architect",
        "timeoutSeconds": 90
    })
    ctx.add("architect", "api", api)
    
    # STAGE 3: Implementation (parallel components)
    components = await fanout_workflow([
        f"Generate {comp} component for {spec['name']} based on:\n{api['output']}"
        for comp in ["handlers", "models", "database", "config"]
    ], agent_id="coder", timeout=180)
    
    if components["success_rate"] < 0.8:
        return {"error": "Implementation incomplete", "retry": True}
    
    # STAGE 4: Tests
    tests = await sessions_spawn({
        "task": f"Generate unit and integration tests for:\n{combine_code(components['results'])}",
        "agentId": "coder",
        "timeoutSeconds": 120
    })
    
    # STAGE 5: Documentation
    docs = await sessions_spawn({
        "task": f"Generate README and deployment guide for {spec['name']}",
        "agentId": "tech-writer",
        "timeoutSeconds": 60
    })
    
    # Write all outputs
    service_dir = f"./services/{spec['name']}"
    await write_outputs(service_dir, {
        "architecture.md": arch["output"],
        "api.yaml": api["output"],
        "src/": components["results"],
        "tests/": tests["output"],
        "README.md": docs["output"]
    })
    
    return {
        "service": spec["name"],
        "path": service_dir,
        "components": 4,
        "tests": True,
        "docs": True,
        "ready_for_deployment": True
    }

# Execution
result = await generate_microservice({
    "name": "user-service",
    "requirements": "User authentication and profile management"
})
# Generates: Complete service with architecture, API, code, tests, docs
```

**Key Patterns**:
- 5-stage pipeline with checkpoints
- Circuit breaker on critical architecture stage
- Parallel component implementation
- 80% success gate before test generation
- Complete artifact generation

## Case Study 6: Data Processing Pipeline

**Goal**: Process large dataset with validation and error recovery.

**Workflow**: Shard → Process → Validate → Merge → Report

```python
async def process_dataset(dataset_path, batch_size=1000):
    """Large-scale data processing with fault tolerance."""
    
    recovery = ErrorRecovery(max_retries=2)
    
    # SHARD: Split into manageable chunks
    shards = await sessions_spawn({
        "task": f"Split {dataset_path} into {batch_size}-record shards, return shard list",
        "agentId": "data-engineer",
        "timeoutSeconds": 60
    })
    
    shard_list = parse_shards(shards["output"])
    
    # PROCESS: Map-Reduce with error recovery
    processed = []
    for shard in shard_list:
        result = await recovery.execute_with_recovery(
            process_shard,
            shard,
            fallback_fn=process_shard_simple  # Fallback if primary fails
        )
        processed.append(result)
    
    # VALIDATE: Check data quality
    quality_checks = await fanout_workflow([
        f"Validate data quality: completeness for {shard}"
        for shard in shard_list[:5]  # Sample validation
    ], agent_id="data-quality", timeout=60)
    
    if quality_checks["success_rate"] < 0.9:
        return {"error": "Data quality insufficient", "review_required": True}
    
    # MERGE: Combine results
    merged = await sessions_spawn({
        "task": f"Merge {len(processed)} processed shards into final output",
        "agentId": "data-engineer",
        "timeoutSeconds": 120
    })
    
    # REPORT: Generate processing summary
    report = await sessions_spawn({
        "task": f"Generate processing report: {len(shard_list)} shards, {len(processed)} processed",
        "agentId": "analyst",
        "timeoutSeconds": 60
    })
    
    return {
        "shards_total": len(shard_list),
        "shards_processed": len(processed),
        "quality_score": quality_checks["success_rate"],
        "output": merged["output"],
        "report": report["output"]
    }

async def process_shard(shard_path):
    """Process single shard with circuit breaker."""
    breaker = CircuitBreaker(failure_threshold=1)  # Strict for data processing
    return await breaker.call(
        sessions_spawn,
        {"task": f"Process data shard: {shard_path}", "agentId": "data-engineer", "timeoutSeconds": 120}
    )

# Execution
result = await process_dataset("./data/large-dataset.csv", batch_size=1000)
# Returns: Processed dataset with quality validation and report
```

**Key Patterns**:
- Sharding for scale
- Per-shard error recovery with fallback
- Sample-based quality validation (5 shards)
- 90% quality gate
- Detailed processing report

## Common Patterns Across Case Studies

| Pattern | Used In | Purpose |
|---------|---------|---------|
| Fan-out | Code review, Research, Data processing | Parallel speed |
| Pipeline | Microservice, Docs | Sequential dependencies |
| Map-Reduce | Docs, Data processing | Scale large tasks |
| Circuit breaker | Microservice, Data processing | Fail fast |
| Error recovery | Data processing | Automatic retry |
| Quality gate | All | Prevent bad outputs |
| Conditional branching | Incident response | Context-aware flow |
| Context tracking | Incident, Microservice | Audit & resume |

---

## Selection Guide

**Use Case → Recommended Pattern**:
- Parallel analysis → Fan-out (Code review, Research)
- Sequential build → Pipeline (Microservice, Docs)
- Large data → Map-Reduce (Data processing)
- Time-critical → Parallel + Fast fail (Incident response)
- High-stakes → Pipeline + Quality gates (Microservice)
