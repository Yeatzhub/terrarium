# Empty States — UX Pattern Documentation

## What
- Screens shown when there's no content to display
- Zero results, empty inbox, unconfigured dashboard, cleared lists
- Opportunity to educate, guide, and reduce abandonment

## The Hierarchy

### 1. Blank (Anti-pattern)
```
[ Just empty space ]
```
- User: "Did it load? Is it broken?"
- Never ship this

### 2. Basic
```
No messages found
```
- Informs but doesn't help
- Minimum viable, minimum useful

### 3. Good (Standard)
```
[ 📭 Icon ]
Your inbox is empty
New messages will appear here when you receive them.
```
- Acknowledges state + explains why + hints at future

### 4. Actionable (Best)
```
[ 📭 Icon ]
Your inbox is empty
Start a conversation to see messages here.
[ + New Message ] [ Browse Contacts ]
```
- + Clear next step to get to non-empty state

## Anatomy

### Required Elements
| Element | Purpose | When to Omit |
|---------|---------|--------------|
| **Icon/Visual** | Recognition, emotional cue | Never |
| **Headline** | State name (what's empty) | Never |
| **Body text** | Why it's empty, what belongs here | If obvious from headline |
| **CTA** | How to populate it | If user can't act (no perms) |
| **Secondary link** | Alternative path | If there's only one action |

### Layout
```
┌─────────────────────────┐
│                         │
│      [   Icon   ]       │
│                         │
│      Empty Headline     │
│    Description text     │
│                         │
│  [ Primary CTA ]        │
│  Secondary text link    │
│                         │
└─────────────────────────┘

- 24px between elements
- Max width: 400px
- Centered in container
- Icon: 48-64px, subdued color
```

## Tone Matrix

| Context | Tone | Example Headline |
|---------|------|------------------|
| User caused it (cleared inbox) | Celebratory | "All caught up! " |
| System result (no matches) | Neutral | "No results found" |
| Set-up needed (new feature) | Encouraging | "Let's set up your dashboard" |
| Missing data (user deleted all) | Empathetic | "Nothing here yet" |
| Error state | Direct | "Couldn't load items" |

## Common Patterns

### Zero Search Results
```
[ 🔍 ]
No results for "xyz"
Try adjusting your filters or search terms.

[ Clear Filters ]  [ Modify Search ]
```

### Empty List (User Actionable)
```
[ 📋 ]
No tasks yet
Create your first task to get started.

[ + Create Task ]
```

### Empty List (Permission Blocked)
```
[ 🔒 ]
No access to tasks
Contact your administrator to request access.

[ Request Access ]
```

### New Feature / Onboarding
```
[ 📊 ]
Analytics ready when you are
Connect your data source to see insights.

[ Connect Data ]  [ Learn More → ]
```

### Error Empty State
```
[ ⚠️ ]
Couldn't load messages
Something went wrong. Try again in a moment.

[ Retry ]
```

## Visual Treatment

### Icons
- Use filled icons, not outline (more substantial)
- Subdued colors: gray-300/400, not black
- Empty state ≠ error icon unless it's an error

### Colors
```css
.empty-state {
  color: var(--text-muted); /* Not primary black */
}

.empty-state__icon {
  color: var(--icon-subdued);
  opacity: 0.6;
}

.empty-state__cta {
  color: var(--text-primary); /* Bring focus to action */
}
```

### Motion (Optional)
- Gentle icon animation on first view (1x only)
- Don't loop infinitely (distracting)
- Respect `prefers-reduced-motion`

## Do / Don't

✅ **Do**
- Explain why it's empty ("No orders YET" vs just "No orders")
- Offer a clear primary action if user can populate it
- Use "you" language ("Your inbox" not "Inbox")
- Consider the emotional state (celebrate cleared todos)

❌ **Don't**
- Say "Error" when it loaded successfully ("No results" ≠ error)
- Use scary red icons for neutral empty states
- Hide empty states behind loading spinners (flash of content)
- Offer actions user can't take (permissions, paid features)

## Accessibility

```html
<div class="empty-state" role="status" aria-live="polite">
  <svg aria-hidden="true">...icon...</svg>
  <h2>Your inbox is empty</h2>
  <p>New messages will appear here.</p>
  <button>New Message</button>
</div>
```

- `role="status"` for polite announcement
- Don't trap focus in empty state (actionable, not modal)
- Keyboard accessible CTAs

## Empty State Flow

```
Loading → Empty State → Content
   │           │            │
   │           │            └─ Success: populated list
   │           │
   │           └─ Explain, offer CTA
   │
   └─ Skeleton or spinner (brief)
```

Show skeleton/loading only briefly (300-500ms max), then empty state. Never show "No items" while still loading.

## Responsive

```css
.empty-state {
  padding: 2rem;
  text-align: center;
}

@media (max-width: 400px) {
  .empty-state {
    padding: 1.5rem 1rem;
  }
  .empty-state__cta {
    width: 100%;
  }
}
```

## Metrics to Watch
- Time to first action (from empty to CTA click)
- Empty state → conversion rate
- Support tickets asking "why is X empty?"

## One-Liner
Empty states are opportunities, not dead ends — explain the state, celebrate progress (cleared lists), and always offer a clear path forward with a primary CTA.
