# Graphics Requirements Template

Generated by Brokkr when checking Android projects for missing assets.

## Location
Each Android project has a `graphics-requirements.json` in its assets folder:
```
/storage/workspace/projects/android/{app}/assets/graphics-requirements.json
```

## Schema

```json
{
  "app": "terrarium",
  "last_check": "2026-03-25T22:30:00Z",
  "required": [
    {
      "type": "icon",
      "name": "launcher_icon",
      "sizes": ["mdpi", "hdpi", "xhdpi", "xxhdpi", "xxxhdpi"],
      "dimensions": [48, 72, 96, 144, 192],
      "status": "complete",
      "path": "icons/mipmap-{density}/icon.png"
    },
    {
      "type": "banner",
      "name": "play_store_banner",
      "sizes": ["feature"],
      "dimensions": [1024, 500],
      "status": "missing",
      "path": "banners/play_store.png"
    },
    {
      "type": "splash",
      "name": "launch_screen",
      "sizes": ["default"],
      "dimensions": [1080, 1920],
      "status": "missing",
      "path": "splash/splash.png"
    }
  ],
  "pending": ["play_store_banner", "launch_screen"]
}
```

## Status Values
- `complete` - Asset exists at correct path
- `missing` - Asset needed, not yet generated
- `pending_approval` - Bragi generated, awaiting user approval
- `approved` - User approved, ready for Brokkr to include

## Asset Types

| Type | Common Sizes | Usage |
|------|-------------|-------|
| icon | 48-192px square | App launcher, notification |
| banner | 1024x500px | Play Store feature |
| splash | 1080x1920px | Launch screen |
| notification | 24-96px square | Push notification icon |
| adaptive_icon | 108dp circle | Android 8+ adaptive icon |

## Workflow

1. **Brokkr checks project** → generates/updates requirements.json
2. **If pending items** → Brokkr posts to #graphics: "Bragi needed: [list]"
3. **Bragi generates** → posts variations to #graphics
4. **User approves** → status → "approved"
5. **Brokkr includes** → copies assets to build on next deployment
